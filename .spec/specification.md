# Specification: 公文 AI Agent (MVP)

## 1. Overview
打造一套「以真實台灣政府公文為黃金標準、由多個 AI Agent 互相審查監督」的公文撰擬系統，協助承辦人與公文寫作學習者，在數分鐘內產出符合《文書處理手冊》與「政府文書格式參考規範」的高品質公文草稿。

## 2. User Stories
- **US-01 需求分析**: 作為承辦人，我希望能透過對話輸入發文需求（如案由、對象、金額），讓 AI 自動整理成結構化的公文需求表單。
- **US-02 範例檢索**: 作為承辦人，我希望系統能從過往公文或範本庫中，找出最相似的參考範例，以確保用語符合機關慣例。
- **US-03 草稿生成**: 作為承辦人，我希望能一鍵生成符合政府格式標準（字號、主旨、說明、辦法）的公文初稿。
- **US-04 多重審查**: 作為主管，我希望 AI 能自動檢查格式錯誤、法規引用風險與用語不當之處，並提供審查報告。
- **US-05 格式匯出**: 作為使用者，我希望能將定稿匯出為標準 Word (.docx) 格式，以便直接列印或發文。

## 3. Acceptance Criteria
- **AC-01**: 系統能正確識別「函」、「公告」、「簽」三種核心公文類型。
- **AC-02**: 輸出的公文格式必須 100% 符合《文書處理手冊》定義的欄位結構（主旨、說明、辦法）。
- **AC-03**: 多 Agent 審查機制必須包含至少 4 個維度：格式稽核、文風檢查、事實查核、一致性檢查。
- **AC-04**: 系統必須為每個重要的內容生成提供「來源依據」（如引用法條或參考範例 ID）。
- **AC-05**: CLI 命令 `gov-ai generate` 能在 60 秒內完成從需求到審查後的草稿生成。

## 4. Constraints
- **本地優先**: 知識庫 (Chroma) 與配置必須支援本地部署。
- **CLI-First**: 所有核心功能必須透過 CLI 介面暴露，支援 Unix Pipeline。
- **Python**: 專案主語言為 Python 3.11+。
